Case Study: SentinelOS

Problem

Student data was at risk due to weak authentication and unencrypted storage. Academic environments needed a secure, modular desktop OS.

Solution

SentinelOS was built with robust user authentication, encrypted file storage, and system monitoring. The architecture prioritized security and reliability.

Architecture

  • UI → Core Services → Security Module → Database
  • Encrypted file storage
  • System monitoring

Tech Stack

  • C#, .NET, SQL Server, WinForms

Database Schema

CREATE TABLE Users (
    id INT PRIMARY KEY,
    username VARCHAR(50),
    password_hash VARCHAR(255),
    role VARCHAR(20)
);
CREATE TABLE Files (
    id INT PRIMARY KEY,
    owner_id INT,
    filename VARCHAR(100),
    encrypted_data BLOB
);

Challenges

  • Ensuring robust encryption
  • Balancing usability and security
  • Handling system monitoring efficiently

Performance Considerations

  • Optimized for academic workloads
  • Low resource usage

Refactor Ideas

  • Upgrade encryption algorithms
  • Improve modularity for plugins

Impact

Improved data safety, reduced breaches, and increased system reliability for 200+ users.
